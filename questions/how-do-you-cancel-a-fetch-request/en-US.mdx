## TL;DR

To cancel a `fetch` request, you can use the `AbortController` API. Create an instance of `AbortController`, pass its `signal` property to the `fetch` request, and call the `abort` method on the controller to cancel the request.

```js
const controller = new AbortController();
const signal = controller.signal;

fetch('https://example.com', { signal })
  .then((response) => response.json())
  .then((data) => console.log(data))
  .catch((err) => {
    if (err.name === 'AbortError') {
      console.log('Fetch aborted');
    } else {
      console.error('Fetch error:', err);
    }
  });

// To cancel the request
controller.abort();
```

## How do you cancel a `fetch` request?

### Using the `AbortController` API

The `AbortController` API provides a way to abort web requests such as `fetch`. Here's a step-by-step guide on how to use it:

1. **Create an instance of `AbortController`**:

   ```js
   const controller = new AbortController();
   ```

2. **Get the `signal` property from the controller**:

   ```js
   const signal = controller.signal;
   ```

3. **Pass the `signal` to the `fetch` request**:

   ```js
   fetch('https://example.com', { signal })
     .then((response) => response.json())
     .then((data) => console.log(data))
     .catch((err) => {
       if (err.name === 'AbortError') {
         console.log('Fetch aborted');
       } else {
         console.error('Fetch error:', err);
       }
     });
   ```

4. **Call the `abort` method to cancel the request**:
   ```js
   controller.abort();
   ```

### Handling the abort error

When a fetch request is aborted, it throws an error with the name `AbortError`. You can handle this error in the `catch` block of the promise chain:

```js
fetch('https://example.com', { signal })
  .then((response) => response.json())
  .then((data) => console.log(data))
  .catch((err) => {
    if (err.name === 'AbortError') {
      console.log('Fetch aborted');
    } else {
      console.error('Fetch error:', err);
    }
  });
```
