## TL;DR

CommonJS and ES modules are two different module systems in JavaScript. CommonJS is used primarily in Node.js and uses `require` and `module.exports` for importing and exporting modules. ES modules are the standard for JavaScript and use `import` and `export` statements. ES modules are statically analyzed, which allows for better optimization, while CommonJS modules are dynamically loaded.

---

## Difference between CommonJS and ES modules

### CommonJS

CommonJS is a module system used primarily in Node.js. It uses `require` to import modules and `module.exports` to export them.

#### Example

```js
// exporting a module in CommonJS
const myModule = {
  name: 'CommonJS Module',
  greet: function () {
    console.log('Hello from CommonJS');
  },
};

module.exports = myModule;

// importing a module in CommonJS
const importedModule = require('./myModule');
importedModule.greet(); // Output: Hello from CommonJS
```

### ES modules

ES modules are the standard module system for JavaScript, supported natively in modern browsers and Node.js. They use `import` and `export` statements.

#### Example

```js
// exporting a module in ES modules
export const myModule = {
  name: 'ES Module',
  greet() {
    console.log('Hello from ES Module');
  },
};

// importing a module in ES modules
import { myModule } from './myModule.js';
myModule.greet(); // Output: Hello from ES Module
```

### Key differences

#### Syntax

- CommonJS uses `require` and `module.exports`
- ES modules use `import` and `export`

#### Loading

- CommonJS modules are loaded synchronously
- ES modules can be loaded asynchronously

#### Static vs dynamic

- ES modules are statically analyzed, allowing for better optimization and tree-shaking
- CommonJS modules are dynamically loaded, which can be less efficient

#### Scope

- CommonJS modules have their own scope
- ES modules have strict mode enabled by default and do not have their own scope
